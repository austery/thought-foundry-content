### 向上管理的核心

**嘉宾**:
那接下来问你两个具体的，对于想爬梯子的同学们的两个具体的方法问题。第一个就是你的向上管理其实很强。我跟你一起做那个 **AI analytics** 的 **panel**（座谈会）的时候，就发现你跟你的老板副总裁 (VP) 他的沟通，我能感受到你的向上管理能力是很强的。就是他蛮信任你的，你跟他说话就是很明显非常 close（亲近），你有什么事情可以随时给他发东西。这个其实是很多 director 和 VP 之间没有的关系。

向上管理，这个东西当然有很多运气啊，细节这都没办法 **replicate**（复制）。但最重要一点，可能还是一个 **expectation management**（预期管理）的问题。就是说当你有一次机会，别人交给你一个任务，他有足够的能力去 **recognize**（认可）你的 **contribution**（贡献）的时候，你如果可以 **beat their expectation**（超越他们的预期），就是 **show them that you add value to them**（向他们展示你增加了价值），就你可以解决掉他们的问题，而且解决得比其他人漂亮，然后让他们知道你对他们是有价值的。这个其实最重要的东西。

我觉得，就当你可以 **repeatedly showing people**（一次又一次地向人们展示），这些东西你交给我，你放心对吧，我一定漂漂亮亮给你完成，不用管那些乱七八糟的。是人都会 **recognize**（认可）这样一个人在他身边的价值。然后自然而然地，会尽可能的会把类似的问题，甚至就是说可能说“哎，我觉得这个问题稍微有点挑战性，但是这个人很棒，我要交给他”。对，这个我觉得最最重要一点在这里。

好，竟然你会选这个是最重要一点，我觉得非常 make sense。我给大家解读一下，你自己不知道你的 **superpower**（超能力）在这儿，但是我看了足够多在这方面没做好的同学和我自己，我知道这个东西难点在哪儿了。难点在于，你想做成任何一件这种 **beat expectation**（超越预期）的事情，你一定有很多脏活累活杂活，你一定要非常紧密地 **follow up**（跟进）。你要真的去 **care**（关心）那个结果。很多人呢，他要么就是选择那个有 **intellectual**（智力）挑战的事情，他去做那个难的事情，但是那些脏活累活他不做，他没有 **go the last mile**（走完最后一公里）。其实这件事情就等于没做，或者说就效果大打折扣。

所以说你就是，第一你要真的对这个结果是关心的，然后第二，你是愿意去不厌其烦地去 **follow up**（跟进）这些东西。这个在很多人看起来是低价值工作，但是其实它是为了达成这个结果来说非常必要的。就是这种，我把它叫做 **project management**（项目管理）工作吧。一个好的 **project manager**（项目经理）和不好的 **project manager**（项目经理），对这件事情 **deliver**（交付）的是不是漂亮，其实影响很大。

### 深入理解用户需求

**嘉宾**:
其实这点，我觉得所有人都是有这个能力的。因为你自己有这个 **superpower**（超能力），你觉得就对别人来说是很简单的。其实不是。我试着 **argue back**（反驳）一点。呃不是，I mean... 对对对。我觉得就是说你看什么事情，就比如说任何一个人租一套房子的时候，他会去干什么？他会去货比三家，他会去看，哎，我这边这个位置怎么样，窗外吵不吵，有没有飞机，有没有火车，然后 **neighbor**（邻居）什么样对不对？我就不会。

证明这个东西你是不 **care**（关心）的。我知道很多很多的人，遇到他真正 **care**（关心）的东西的时候，他都知道什么是重要的，哪些细节我要注意，因为他知道这个东西如果他没做好，最后倒霉的是他自己，因为他是 **the end user at the end**（最终的终端用户）。所以说其实工作中很多时候就是，我说的就是怎么去 **impress**（打动）别人，那就是 **将心比心**。把自己当做，哦，这个人他交给我这件事情，他一定是 **care**（关心）的。那我怎么把我自己当做是他，或者说这个东西最后做完了以后，我最倒霉的情况下，我应该怎么样去尽可能的把这个事情做好。上心一定要上心。

### 提升职场存在感

**嘉宾**:
第二个就是，你似乎能非常好地 **register into a conversation**（加入一段对话）。当你同组的人做一件你想掺和的事情，比如说你觉得掺和了有好处，他们能达到一个你想达到的结果，你能从中获益。那这个时候有很多种参与方式。如果处理不好呢，别人就会说，你是不是来抢我的 **credit**（功劳）？进去以后也插不进嘴。就是你似乎有这样的能力，就是你既不显得在抢别人东西，但是你又能让你的存在感足够强。

我觉得可能分两个方面来讲。第一个就是说，你需要有一定的时间和精力来投入这个问题，这可能事先要做准备，对吧？你不能让自己特别特别忙，或者你需要有一定的东西，你可以 **prioritize**（划分优先级），然后你这样有时间去做这个事情。那为什么要有时间呢？这就是第二点，就是说你加入一个 **workstream**（工作流）的时候，如何让已经在这个 **workstream**（工作流）上的人觉得，就是说他不是来抢地方或者怎么样。最简单的就是 **provide value from the get-go**（从一开始就提供价值）。比如说他们懒得做的东西，这些脏活累活，你去做了。然后他们有的这个没想到的东西，你帮人家想了。大家就会欢迎。

这样的人，他再不喜欢让你参与进来，他不想跟你分这个 **credit**（功劳），但是你讲的有道理，他听了以后对他自己也是有好处的，所以他就会听。那这个为什么会需要第一点呢？就是你需要花时间去想，你需要花时间去 **collect the context**（收集背景信息），知道他们现在都做了什么，然后你自己仔细去想，你就往往会发现，欸，这个里面有哪些可以改进的。因为我相信任何一个项目，任何一个产品，一定有可以继续改进的点，最多是大小的问题。找到这个点，提出来。很大的点，可能他们自己会做。很小的点，那你先做，不要等别人。然后这样的话，大家就会觉得，哎，这个人过来是帮我们的。

怎么样子真的做到将心比心？因为你刚刚说的这个 **provide value**（提供价值），是吧？很多人是出自自己的一厢情愿。这句话可能有点老土，就是从用户的角度出发吧，这个 **mindset**（思维模式）可能是大家都有的。难点就在于，就是我们怎么知道用户究竟想要什么。但其实呢，我觉得在公司内部的项目，往往没有公司外部就直接对 user 的那么难。因为 user 他并不一定会真的告诉你，一个人告诉你，不代表一百万个人都是这么想。但是公司内部的项目，往往是有真实的 user 存在的，他就那么几个 user，就假设是隔壁组的 VP 吧。这你就可以去问他。

如果我看到以前啊，大家是这么做的，然后我不去问 user，很多时候我就会产生一种想法，就是“哦，大家以前都这么做，那我就这么做”。就我到一个新的组，我是一个 **analytics**（分析）组，这组没有 **dashboard**（仪表盘）啊，**build some dashboards**（构建仪表盘）。这组没有那个好的 **data foundation**（数据基础），**build some data foundation**（构建数据基础）。这组没有好的 **metrics**（指标），那就 **design some metrics**（设计一些指标）。这就有一点把之前别的 user 喜欢的东西套用到现在的 user 身上的这样的一个行为。那我觉得这个行为其实是很多时候不一定是对的。有很多时候它有可能是对的，但是你不如直接去跟 user 聊一下。

尤其在公司内，你这么做的时候你怎么开头？我大概会就讲一个具体的情况，我就比如说，说“哎，那天开会的时候，我看到有这个问题，然后你这么问了，然后我好像觉得大家的回答要不然是没有特别到点上，要不然你好像没有 **satisfied**（满意）”。然后我猜测大概是这个样子的，你觉得呢？”然后他说“知音！”不一定，有的时候他也不一定跟你讲……“傻逼！”对，这个这个对。但反正你问了也没有什么，也不会怎么样对吧？你最多他就会觉得这个人啊有点心思，但是好像这个智商还有待开发啊。所以没达到点子上，但是他不会怪你啊。最多你就是 **lose out a chance to impress people**（失去一个给别人留下深刻印象的机会）。但实际上这个……无所谓。你不说也没有那个 chance（机会）对啊。

如果你把所有你认为对这个项目了解的人都问了一圈，发现大家就是没有很好的答案，恭喜你，你可能发现了一个非常非常好的问题。这个时候你再去拿出去问，我觉得就很好。

### 时间管理与优先级

**嘉宾**:
好的，那你怎么样子空出来时间？因为你自己作为一个 director，是吧，你也很忙，就是你怎么可以保证有时间的？我觉得可能说到底也只能是去 **prioritize**（划分优先级）。Like what？是大家觉得重要，其实不重要的东西。我自己的想法，就是说还是从这个事情本身的 user impact 的角度来讲。如果这件事情没有人做，what would happen？然后如果这个事情很好地做出来了，what would happen？然后如果有的东西你可以等一下，即使他有可能是你老板交给你的事情，然后你想了一下，发现其实这个东西哎，它不一定有那么重要。当然，你可能要先跟你自己老板确认一下啊。那这些东西可能就是你可以推一下，可以 **drop**（放弃）一下，可以往后拖一拖的东西。

然后呢，我觉得还有一点，就是说有的时候，你觉得自己已经有很满的 **plate**（任务）了，但是他又有一个新的东西进来。那这个时候你也要想办法去自己去 **assess**（评估）新东西进来以后，它在你的这个系统里面，在你这个世界里面，它的 **stack rank**（排序）到底是什么样的。如果它是真的是很有价值，很有意义的东西，只能恭喜你，你找到了更有意义的问题。那请从你之前那些 less 有意义的问题里面，找一些丢掉它。也只能这样子。

### 构建个人核心竞争力

**嘉宾**:
就像我觉得你做的这种别人做不了的事情，你做出来了，并且你有办法把它系统化，沉淀成自己的一个技能，以后你还能做。这个就是一个别人取代不了的东西。公司有一天可以没有，但别人做不了的东西，假设它还是有意义的，那它就是很 **valuable**（有价值）的东西。这个东西怎么别人怎么拿都拿不走。所以某种程度上来讲，这反而是对于个人，长期来讲，我觉得更重要一点。

那我觉得这个就可以 **call back**（呼应）一下之前那个点，就是 **play your own game**（玩你自己的游戏）。找到你自己认为有意义、有价值的东西。那在你自己的这个 game 里面，就是没有人可以击败你。所以这个点上就是我相信是，除非你做的事情真是对所有人，对所有社会一点意义都没有，而且是 **negative**（负面）的，那 **you should stop it right away**（你应该立刻停止）。但是假设他是没有那么糟糕，然后又你自己认为有趣，那我觉得这就足够了。

### 玩自己的游戏

**嘉宾**:
插个很 random 的题外话啊。最近看了一个电影叫 **《Ballerina》**（《芭蕾女杀手》）。因为那个主人公是个女生，做杀手嘛。有一段小故事，就是她怎么打都打不过她那些男生的同学们。然后后来教练就跟她说，“you need to fight like a girl”（你要像个女孩一样去战斗）。什么是 **fight like a girl**（像女孩一样战斗）？他那个电影里面讲的很白痴，就是她专门去攻击男生的这个弱点，生理弱点啊，很白痴。但实际上，这个跟那个 **play your own game**（玩你自己的游戏）就是一模一样的东西。不要拿别人的这种已经定性好的，什么是 **right thing**（正确的事情），什么是 **success**（成功），什么是 **success** 的标杆来要求自己。应该说，**open to accept an alternative world**（对另一个世界敞开怀抱），然后用自己的这个…… **definition**（定义）去说，在这个 game 里面我定了规则，那我只需要好好做就可以了。可能这个胜利对于别人来讲没有什么意义，**by the way**（顺带一提），但是对于你自己有意义，我觉得这个就很重要。
