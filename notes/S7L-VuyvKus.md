### 使命驱动：从 Cursor 回归 Anthropic 的安全底层
全球 4% 的 **GitHub** 代码提交已由 AI 完成，顶尖工程师自去年 12 月以来甚至没写过一行代码。这一变革背后的核心推手是 **Claude Code** 及其核心开发者 **鲍里斯·切尔尼**（Boris Cherny）。鲍里斯曾任职于 **Meta** 专注于工程效率研究，后加入 **Anthropic** 领导开发了如今 AI 编程领域的标杆工具。

在一段引人深思的职业插曲中，鲍里斯曾短暂离开 Anthropic 加入 **Cursor** 团队，却在两周后毅然回归。他坦言，这种选择的核心在于 Anthropic 深入骨髓的 **安全使命**。在 Anthropic，无论构建多么酷炫的产品，都无法替代对安全性的极致追求所带来的使命感。这种驱动力不仅是吸引他回归的根本，也贯穿了 Claude Code 从诞生到爆发的全过程。如今，Claude Code 已发布一周年，仅过去一个月的日活跃用户数就翻了一番，预计到今年年底，GitHub 上五分之一的代码提交将由其完成。

### 编程的终结：从写代码到“构建者”的跨越
鲍里斯提出了一个震撼行业的观点：**编码本身已经被解决了**。自去年 11 月以来，他个人 100% 的代码都由 Claude Code 编写，每天提交 10 到 30 个变更请求，却从未亲手编辑过一行代码。在他看来，工程师的核心乐趣已从繁琐的语法编写和调试细节，转向了构思产品逻辑、设计系统架构以及与用户沟通需求。

他预测，**软件工程师**（Software Engineer）这个头衔将逐渐消失，取而代之的是 **构建者**（Builder）。这一转变被他比作 **印刷机** 的发明：14世纪的欧洲识字率极低，书写被抄写员垄断；印刷机的出现让知识民主化，催生了文艺复兴。如今，AI 编程工具就是现代的印刷机，它打破了编程技能的垄断，让每个人都具备构建软件的能力。鲍里斯本人是自学编程出身，最初是为了在数学考试中“作弊”而编写小程序，这段经历让他深刻意识到：编程从来不是目的，而是解决问题的工具。

### Agentic AI 崛起：Cowork 与行动导向的未来
变革的下一步正在从编程扩展到所有基于电脑的工作，即 **智能体 AI**（Agentic AI：能够使用工具、控制电脑并在现实世界执行任务的 AI 系统）。鲍里斯团队推出的 **Cowork** 产品正是这一趋势的典型代表。该产品的诞生源于对用户“滥用”Claude Code 的观察——非技术人员开始用它做 SQL 分析、基因组研究甚至恢复损坏硬盘中的照片。

基于这些潜在需求，团队仅用 10 天就构建了面向大众的桌面 Agent。现在的 Cowork 已能自动处理项目管理、同步电子表格、收发邮件，甚至通过 **Chrome 集成** 登录系统填写复杂的医疗表格 PDF。鲍里斯认为，Cowork 只是一个开始，未来任何需要使用计算机工具的职业都将面临这场变革。这也呼应了 Anthropic 的产品哲学：不要被当前模型的能力限制想象力，而应 **面向未来 6 个月的模型进行构建**。这种“押注未来”的策略确保了当新一代模型发布时，产品能立即实现指数级增长。

### 苦涩的教训：赋予模型工具而非僵化流程
在开发方法论上，鲍里斯推崇 **Rich Sutton** 提出的 **苦涩的教训**（The Bitter Lesson: 长期来看，通用的计算和搜索方法总是胜过利用人类知识构建的复杂专用逻辑）。他强调，不要给模型搭建过多的脚手架或强制性的“第一步、第二步”工作流。最好的方式是赋予模型工具和目标，让其自主规划执行路径。随着模型能力的提升，过多的限制反而会束缚 AI 的潜力，因为脚手架带来的微小性能提升，终会被下一代通用模型的能力所抹平。

对于企业和开发者，他分享了极具价值的实操建议。在项目初期，不应控制工程师的 **Token 预算**，而应给予无限的自由度去创新，因为此时的 Token 成本相对于人力成本微不足道。对于个人开发者，他提出了三个技巧：始终使用能力最强的模型（如 **Opus**），善用 **计划模式**（Plan Mode：在提示词中加入“先不要写任何代码”以讨论方案），并尝试多种交互界面（如 Slack 集成或 IDE 扩展）以寻找最适合的工作流。

### 安全三层架构：力争上游的开源精神
即便在追求极致效率的同时，Anthropic 依然将安全置于首位。他们建立了三层安全架构：
1. **机制可解释性**（Mechanistic Interpretability: 深入理解神经网络内部运作，追踪特定神经元的激活）。
2. **实验室评估**：在合成情景中测试模型对齐情况。
3. **尽早发布产品**：在真实应用中观察行为并持续优化。

为了推动行业共同进步，他们发布了开源的 **沙盒环境**，确保 Agent 在边界限制内运行。这种“力争上游”的原则旨在让更多企业能安全地应用 Agent 技术。鲍里斯预测，AI 的冲击将首先模糊产品经理、设计师与工程师的边界，**通才** 将成为未来最核心的竞争力。在 Anthropic，设计师大多会写代码，这种角色边界的模糊化正在重塑人才市场。

### 杰文斯悖论：技术进步创造的更多可能
面对 AI 导致失业的担忧，鲍里斯引用了 **杰文斯悖论**（Jevons Paradox: 提高资源利用效率不仅不会降低需求，反而会因为使用成本降低而导致总需求增加）。印刷机的出现并没有让抄写员失业，而是解放了他们的时间去从事更有创造性的书籍插图工作，并催生了庞大的出版业。AI 成为工程核心后，Anthropic 的团队规模反而持续扩张，这证明了知识民主化将带来更多新的职业机会。

在个人生活中，鲍里斯表现出对长周期事物的迷恋。他深受中国科幻作家 **刘慈欣** 的影响，认为《流浪地球》短篇集展现了独特的中国科幻视角。这种对宏大叙事的思考也影响了他的职业选择。当被问及实现 **通用人工智能**（AGI）后想做什么时，他的答案是去日本乡村 **做味噌**。这种需要漫长时间等待（红味噌需两三年）的手工劳动，是他对抗快节奏工程工作的平衡点。他的格言简单而深刻：**运用常识**。真正优秀的结果往往来自那些从第一性原理思考、不机械遵循流程的人。

在鲍里斯眼中，这场 AI 变革本质上是在推动人类创造力的最大化。与其恐惧被取代，不如主动学习工具，培养跨学科思维，从繁琐劳动中解脱，去专注于那些更具价值的创造性工作。
