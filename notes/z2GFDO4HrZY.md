### OpenCode 的野蛮生长
在这波 **AI 编程工具**（AI Programming Tools: 利用大语言模型辅助编码的开发辅助软件）的浪潮里，OpenCode 的增长速度堪称恐怖。几个月前其月活只有 65 万，现在已冲到了 650 万，更有望突破 800 万。其联合创始人 **达克斯·拉德**（Dax Raad）在播客访谈中指出，当所有 AI 工具创始人都在吹嘘模型有多聪明、生产力能翻十倍时，他不仅没有鼓吹 AI 万能，反而一针见血地指出了当前工具在真实工程团队中制造的三个致命错觉。

拉德的编程之路始于《我的世界》（Minecraft）社区，他通过编写大量 Java 代码构建沙盒情境，这不仅提升了技术，还培养了他观察系统内用户行为的视角。在经历了 Ride Health 的创业挑战后，他在 SST 和 OpenNext 的开源实践中积累了深厚的开发者服务经验。OpenCode 的诞生，源于团队在盈亏平衡后的战略思考：在这个十年，开发者工具无法回避 AI。经过团队对 Claude Code 的深度试用，他们决定自己打造更符合工作流的编程工具。OpenCode 上线后的爆炸性增长，部分受益于季节性因素，更关键的是 Anthropic 对其订阅接入的封禁引发了社区激烈反响，这促使 OpenCode 迅速联合了其他模型厂商，成功塑造了“中立、开放”的生态位。

### AI 编程的三个致命错觉
为什么在市场如此火热时，开源中立的 **多模型编程智能体**（Multi-model Coding Agent: 支持接入多种底层大语言模型的自动编程代理）位置仍为空白？拉德认为，开发者工具领域最大的结构性优势在于开发者本身，但痛点在于程序员往往不擅长做 B2C 产品。他主张必须像做消费级产品一样思考它，极度重视用户首次打开时的感受，甚至底层采用了自研的终端渲染框架。

在此基础上，拉德直指当前 AI 编码存在的三个致命错觉：

1. **AI 自动加速论的虚幻**: 很多人以为 AI 是“按钮”，按一下效率就翻倍。事实是，在一个激励平庸的组织里，大多数人只是利用 AI 更快完成同样的工作量，并将省下的精力留给自己。
2. **生产力增量的误判**: 即使开发执行层有所加快，软件工程的瓶颈依然在于明确“做什么”。在拉德看来，AI 也许提高了 20% 的执行效率，但世界并未变得轻松十倍。
3. **“AI 必胜”的心理防御**: 社交媒体上充斥着关于 AI 时代的宏大预测，这本质上是人们在剧烈变化中的心理防御机制，通过预言一个自己会赢的未来来缓解焦虑。

### 重构开发者工具的 B2C 思维
面对企业 CFO 对工程师额外成本的质疑，拉德冷静地指出，技术推广会经历炫耀期，但随后会面临真正的财务考验。如果无法证明产出显著增加，单纯的高额 AI 预算难以持续。对于工程领导者而言，AI 的净结果可能只是让工程师更开心（工作更轻松），但这对很多追求高产出的公司而言依然不够。

针对这种现状，拉德向团队发出的备忘录强调了质量的重要性：发布无意义的功能、往错误设计上叠补丁，并不会换来真正的速度。他警告，在高速奔跑的幻觉中，最容易骗过的人就是自己。只有重新相信慢一点、把地基打牢，才是最快的路。他极度推崇 **领域驱动设计**（Domain-Driven Design: 一种通过将实现连接到核心业务概念来构建软件的方法），认为这种曾被高手嫌弃为“繁琐”的旧模式，在 Coding Agent 时代因为不需要人手打字，其在可靠性、模块化和安全边界方面的优势被重新放大。

### 品味与质量的非理性坚持
拉德认为，**品味**（Taste: 对产品架构、交互细节及质量标准的卓越判断力）是区分伟大产品的关键，这不仅仅是嘴上说说，更需要做出看似“非理性”的决定。在当前行业内弥漫着“只要商业能赢，代码不必好”的风气下，这种对质量的纵容会如感染般扩散。

他欣赏 HashiCorp 的米切尔·桥本（Mitchell Hashimoto），认同做产品不是单纯塞功能，而是思考新旧功能间的交互。OpenCode 之所以能从 Claude Code 那里抢占用户，正是因为在自研终端渲染框架等不性感、甚至按传统教科书说是“不该做”的事情上投入了巨大精力，从而换来了更稳、更顺的体验。

### 领域专家：工程师的破局点
最后，拉德给资深工程师的职业建议是：软件工程能力是可迁移的技能，但如果你能同时成为某个行业的专家，这个组合将具有极强的稀缺性。大多数工程师太容易把自己变成纯粹的“工单处理者”，加加 UI、改改接口，而不去理解身处的行业。每深入待一年，你对该行业业务的理解就可能超越 99% 的人，这才是 AI 时代工程师最值得抓住的机会。
