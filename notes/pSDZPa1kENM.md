### 人类是思维带宽的瓶颈

**[Jacky]**: 今天的观众来到这个 Workshop 能带走什么呢？首先是提升对 Agent 的认知，原来 Agent 还能这么用；第二个是学会一整套用 AI Agent 做冷邮件的系统；第三个是一套可复用的资产，我准备了一个文件夹，里面有怎么 Set up 这个系统和使用的指南。

**[Indigo]**: 我觉得现在用 AI 的时候，我个人就是瓶颈。因为我还是习惯最多同时开两到三个 Agent 做事，再多我就协调不过来了。我工作的时候，后面挂着 **Claude** 的模型，所以我现在用 **Claude Code** 是主力。我桌上有两台 **Mac mini**，一台专门跑 Session，二十四小时开着。我觉得只要你的 Working Folder 说明书写得好，完全能发挥出威力。我现在的习惯是开一个 **ChatGPT**，开一个 Claude，再用一个 Markdown 编辑器，三个需求分散在不同地方，我很难在一个界面里看这么多 Agent。

**[Jacky]**: 模型能力越来越强，以后只会更强。当你给模型一个任务，只要 **Prompt** 正确，它可以运行三十分钟。中间我要干嘛？刷手机吗？一个很自然的解决方案是并行运行多个 Agent。我就从过程管理者转变成了结果审核者。

我之前在 Terminal 窗口里开七八个 Claude Code，但我发现安排完任务后，之前的就在做什么我就忘记了。瓶颈不在于模型，而在于人类的思维带宽。人的短期记忆只能记住“4 加减 1”个任务。

**[Indigo]**: 所以我不建议大家多线程做事。专注在一个任务里很容易进入**心流**。溜达、洗澡、散步的时候最适合产生创意，专注的工作交给 Agent 就好了，它们比我专注多了。

### Slock：Agent 是一等公民的协作平台

**[Jacky]**: 既然我有七八个窗口，心智带宽不够了，解决方案就是“群聊”。我们把多个 Agent 拉到一个群里，你完成了就回复我。这个我最近一直在用的平台叫 **Slock**。它有点像 **Slack**，但 Agent 是一等公民。

虽然很多大厂在产品里加 Chatbot，但体验远不如从零围绕 Agent 构建的产品。在 Slock 里，你的同事全是 Agent，它们可以读写文件、收发邮件，操作电脑上的所有工具。

**[Indigo]**: 你观察过它们在群里聊什么吗？

**[Jacky]**: 老实说，很多时候是在浪费 **Token**，聊一些不相关的东西。但如果你以特定形式组织它们，就会有生产力。Slock 底层是在平台上抽象了一层后台服务，负责 Agent 之间的频道通讯。下面每一个 Agent 都是一个进程。我还会做一个 **Shared Memory**，在 iCloud 文件夹里给所有 Agent 共享上下文。我把它部署在 Mac mini 上二十四小时运行，发完指令关上电脑去吃饭，体验大概是这样。

### 实战案例 1：自动化冷邮件获取赞助

**[Jacky]**: 讲一个真实案例。这个月我要为 Cursor 组织一个黑客松，来了一百五十多个人。我需要搞一些赞助，虽然官方给钱，但我希望能办得更好。我想尝试发冷邮件。

我的工作流是：
1. **发现与研究**: 我跟 Agent 说我想要 YC (Y Combinator) 的 Startup 赞助，让它去爬 YC Directory 近三年的四百多家公司。
2. **过滤**: 按照规则过滤，比如人数少于一百（流程简单）、To C 产品（需要宣传）、北美地区、AI Agent 相关。最后剩下 77 个联系方式。
3. **深入了解**: 让 Agent 到官网了解每家公司做什，写进 **CSV** 表格。
4. **个性化写作**: 根据信息为每个邮箱写个性化邮件。这如果没有 Agent 是很难完成的。

最后结果：在一个周末发了 77 封邮件，拿到了价值四千美金的赞助，其中很大一部分是现金，还有 **Web Summit** 的门票。

### 冷邮件方法论：进箱率、回复率与发送量

**[Jacky]**: 冷邮件的结果等于：**进箱率 × 回复率 × 发送量**。
*   **进箱率**: 要进 Inbox 而不是 Spam。方法包括：域名隔离（独立域名）、配置 DNS、**Warm-up** (预热邮箱)、控制发送频率（一天 50 封左右，随机间隔）、保持验证（避免退信）。
*   **回复率**: 关键在于**个性化**、**简短**（5 行以内）、有 **Hook** (钩子)、提供价值，以及 **CTA** (**Call to Action**，只问一个低阻力问题）。还有最重要的 **Follow-up** (跟进)。AI 让 Level 4 的深度拥有 Level 0 的成本。
*   **发送量**: 发一封和一千封对 Agent 来说是一样的。

**[Indigo]**: 人做的事情和 AI 做的事情要严格区分。如果一封和一千封没区别，一定让 AI 做。人类应该把时间花在寻找 Surprise 上，把最有价值、难被自动化的事情留给自己。

### 实战案例 2：对抗性编程与信息聚合

**[Jacky]**: 第二个案例是**对抗性编程**。一个 Agent 写代码，另一个独立的 Agent 作为 Reviewer 进行审查，互相对抗，直到 Reviewer 找不出问题。通过这种方式，往往能生成直接可运行的代码。

第三个是**信息聚合**。我有很多信息源：邮箱、Twitter、LinkedIn、IM 平台。我让 Agent 全量读取，过滤噪音，把有用信息在一个频道里展示给我。它还可以帮我回复消息，比如给 X 上的朋友回个消息。

**[Indigo]**: 对于我来说，我还是希望在对应平台上发现惊喜。如果想看病就直说，不用找 Agent 过滤。

### Demo 演示：Agent 团队的日常

**[Jacky]**: 给大家演示一下界面。Slock 就像 Slack，你可以把自己的电脑（比如 Mac mini）连上去。然后创建 Agent Team。我的团队里有：
*   **总经理**: 负责架构和 Memory 调整。
*   **傻福 (Marketing)**: 负责外联和发邮件。
*   **小美 (生活管理员)**: 助理秘书，负责计划行程。
*   **林纳斯 & 泰达**: 软件工程师，负责写代码。
*   **验收工程师**: 负责代码审查。

在冷邮件 Demo 里，总经理 Agent 找公司，傻福 Agent 写 Draft。在对抗性编程演示中，我让它写一个 3D 宇宙游戏。它做完后，Reviewer 找到了几个 Medium/Low 风险的问题，Coder 修复后，游戏就直接可运行了。

**[Indigo]**: 任务还得人类来拆解。太复杂的任务它做不好。我建议技能不要全外包，虽然 AI 能得 80 分，但你让它解释一遍步骤，你自己弄懂架构，这种学习效果最好。

### 总结：从 Prompt 工程到上下文管理

**[Indigo]**: 今天的分享很有启发。Agent 不是软件，而是伙伴。现在的技巧不是 Prompt 工程，而是**上下文管理** (**Context Management**)。为什么 Mac mini 这种本地环境好？因为权限可控，邮件、文件访问都在本地，速度最快，没有云端虚拟机的束缚。

你要给 Agent 工具权限，同时避免被单一平台绑架。我在电脑上写工具手册，所有 Agent 都能调。你的电脑就像一个微型的 **OPC** (**Personal AI Organization**)。

**[Jacky]**: 把公司业务抽象成 Agent 工作流很难，但 SOP 明确的财务、HR、运营流程很容易转化。

**[Indigo]**: 创意的东西还是人来做，人来选。我有个感觉就记在 Note 里发给 Agent，它自己去处理。现在的 AOPC 还在内容阶段，因为产生内容的成本极低。你一个人做一个 SaaS 会很累，但你可以把自己的 SOP 变成 Skill 卖给别人。
