谷歌发布的AI编程助手 **Antigravity** 最近加入了对 **Cloud Opus 4.5** 的支持，这对于许多因 **Claude Code** 封号问题而无法使用 **Cloud Opus 4.5** 平台的用户来说非常有帮助。许多用户在注册 **Cloud** 官方账号后，在 **Cloud Code** 中使用不到一天就会遇到封号问题，而谷歌通过在 **Antigravity** 中集成 **Cloud Opus 4.5**，使得用户能够利用 **Cloud Opus 4.5 Think** 模型执行更复杂的开发任务。

相比其他AI编程工具，**Antigravity** 支持 **多智能体异步工作**，允许在项目开发中同时处理前端与后端，从而大幅提升开发效率。本视频将通过几个代表性案例，展示如何使用 **Antigravity** 中的 **Cloud Opus 4.5** 模型进行项目测试、重构、复杂项目开发等场景，并观察深度使用该模型时是否会遇到额度限制。

### Antigravity 安装与初步设置

**Antigravity** 已全面支持 macOS、Windows 和 Linux 系统。用户只需根据自身操作系统下载并安装即可。安装完成后，首先需要进行初步设置。点击 **Antigravity** 右上角的三个点，选择“自定义”。在“规则”部分，选择“全局”以打开全局规则文件。

在此处，可以填入与技术栈相关的规则文件。例如，针对 Python 开发，可以指定 Python 版本（视频中使用的是 **Python 3.11**），运行命令，创建虚拟环境的方式，以及运行命令和代码的规范，包括代码风格和 **Python 3.11** 的新特性，以及依赖管理命令。

第二部分是设置 **SCSS** 相关的规则文件，包括 **SCSS** 版本（视频中为 **14** 版本），CSS 样式，后端数据库（提及 **Superb**），认证方式，存储方式，强制约束，项目结构和运行命令。完成规则文件设置后，保存并关闭，后续项目开发将遵循这些规则。

### Superb 后端服务配置

接着，再次点击三个点，选择设置 **MCP**。在搜索框输入“**Superb**”并搜索，然后进入 **Superb** 页面。安装 **Superb** 的 **MCP** 后，需要设置 **Token**，这是通过 **MCP** 操作数据库的前提。

**Superb** 可以被理解为开发 App 或网站所需的后端服务。它将原本需要自行搭建和维护的后台功能打包成一个平台，提供数据库服务、登录系统服务、文件存储服务，甚至实时更新服务。使用 **Superb** 可以无需从零搭建后端，快速开发产品并上线，特别适合 **MVP**（Minimum Viable Product）创业原型和小团队的快速迭代。

注册并登录 **Superb** 成功后，点击右上角头像，进入“账户”，在左侧菜单点击“API Token”，然后点击“生成新 Token”。输入一个名称，选择 Token 过期时间（可选择“不过期”），然后点击生成。复制生成的 **Token**，填写到 **Antigravity** 中并保存。

随后，让 **Antigravity** 检查规则文件和 **Superb** **MCP** 是否设置成功。它将运行命令进行检查。检查结果显示 **Superb** **MCP** 连接正常，规则文件加载成功，并且将严格遵守开发规范。

### 测试案例一：项目重构 (旅游规划智能体)

完成设置后，可以开始测试。首先打开一个项目路径，新建一个文件夹并打开。在对话框中选择 **Cloud Opus 4.5 Think** 模型。对话模式下，可以选择“**Fast**”模式处理简单任务，或选择“**Planning**”模式处理复杂任务。此处先选择“**Fast**”模式，测试一个简单的项目重构：将微软智能体框架编写的智能体代码重构为谷歌的 **ADK** 智能体框架。

使用上一次测试 **Codeex** 中 **GPT 5.2** 时的相同提示词，并进行了完整复制粘贴。提示词中包含了阅读谷歌官方文档的链接，要求将微软智能体框架编写的旅游规划智能体重构为谷歌 **ADK** 的智能体框架，并将大模型接口改为 **mAI** 接口，模型设置为 **MLA rge 3**（推测为 **OpenAI API** 和 **GPT-3.5/4** 模型）。同时要求重构后的智能体保持原有逻辑和功能，加入谷歌 **ADK** 的 **UI** 操作界面，并且 API 密钥要求放入 **ENV** 文件。下方附带了微软智能体框架的代码。

运行提示词后，**Antigravity** 开始阅读文档，速度非常快，远超 **Codeex** 的表现。它提示需要创建 **VENV** 虚拟环境，并允许运行安装项目依赖的命令。可以看到其开发速度很快，依赖安装成功后，它开始检查项目结构，并提示项目重构完成。

随后，**Antigravity** 询问是否需要帮助启动 **Web UI** 进行测试。手动设置 **API Key** 后，输入“我已经设置好了 **API**，请为我启动 **Web UI**”并发送。按照提示在浏览器中打开链接，看到了 **UI** 界面。切换到亮色模式，选择创建好的智能体，输入“规划3天的尼泊尔旅行计划”进行测试。

输出结果显示，**Antigravity** 执行的步骤与微软提供的智能体基本一致，包括旅行规划、当地特色活动以及常用语。这表明 **Cloud Opus 4.5** 在 **Antigravity** 中实现的智能体与微软智能体框架的逻辑和步骤保持了一致。相比之下，上次在 **Codeex** 中测试 **GPT 5.2** 时，未能给出正确的代码，而 **Antigravity** 中的 **Cloud Opus 4.5** 效果显著更好。

### 测试案例二：从零开发现代宠物领养平台 MVP

接下来，测试从零开发一个现代化的宠物领养平台 **MVP**。这个测试题之前也用于 **GPT 5.2**，但效果不佳。此次使用相同的题目来测试 **Antigravity** 中的 **Cloud Opus 4.5** 模型。

首先初始化仓库。在规划阶段，选择 **Cloud Opus 4.5 Think** 模型，并使用“**Planning**”模式制定计划，因为项目较为复杂。粘贴提示词后，**Antigravity** 创建了任务，并生成了更具体的计划。随后，选择 **Cloud Opus 4.5** 模型，并点击按钮执行计划。项目规划阶段使用了 **Cloud Opus 4.5**，执行阶段也选择了 **Cloud Opus 4.5**。

项目初始化开始，提示 **Next.js** 已安装，并继续执行命令。左侧文件列表显示生成了相应文件。任务进展到数据库设计，然后执行第三部分任务，速度非常快，远超 **Codeex** 中的 **GPT 5.2**。

等待约 10 分钟后，项目开发完成，生成了项目代码、 **README** 文件，列出了所有功能，以及安装和运行方式。

接下来，输入“让它通过 **Superb MCP** 为我们设置好 **Superb** 数据库”。**README** 文件中给出了数据库迁移方式。调用 **MCP** 自动设置。选择 **Pro** 模型，因为任务相对简单。详细查看了使用 **Pro** 模型完成的完整步骤，并提示数据库设置完成。**Antigravity** 在 **VS Code** 中创建了项目并成功执行了文件。

刷新 **VS Code**，看到了创建的项目。提示无法设置 **ENV** 文件，但给出了文件内容。手动设置 **ENV** 文件后，输入提示词“请为我运行项目”。项目成功启动，显示了美观且功能丰富的首页，包括导航栏、首页、领养宠物入口、领养步骤、登录选项（邮箱、谷歌、 **GitHub** 登录）。

注册账号时，需要验证邮箱，点击确认链接后自动登录成功。查看头像，显示注册邮箱，并有“个人资料”、“领养申请”、“设置”等内容。

以发布者身份测试发布领养信息：上传宠物照片成功，输入宠物名称、种类、品种、年龄（一岁）、性别、体型、毛色、所在地区、健康状况和性格，然后点击发布。发布成功后，点击“浏览领养宠物”，可以看到发布的内容，但图像未成功显示。

通过查看元素并编辑 **HTML**，发现是权限问题导致图像未显示。复制元素内容，输入“发布宠物时上传的图像没有显示”，并粘贴元素内容，发送后 **Antigravity** 成功修复了该报错。刷新后，宠物图像显示清晰，并带有动态效果，整体效果非常不错，远好于 **GPT 5.2** 的实现效果。后续的细节功能可在当前项目基础上继续完善。

### 额度与性能评估

通过这两个案例测试，**Antigravity** 的效果相当不错。使用 **Cloud Opus 4.5** 可以在一定程度上替代 **Cloud Code**。

为了查看 **Cloud Opus 4.5** 的额度是否用完，在新窗口中让其随意生成一段代码，例如智能体的代码。它调用了网络搜索，并输出了多个代码案例及文档链接。尽管之前在 **Antigravity** 中完成了宠物领养平台的开发，**Cloud Opus 4.5** 仍可正常使用，表明其额度非常充足。登录的 **Antigravity** 账户是 **Pro** 账户，使用 **Cloud Opus 4.5** 进行项目开发已完全足够。
