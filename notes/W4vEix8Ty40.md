### 嘉宾自我介绍与学习背景

**Speaker 0**: 今天我们聊的内容啊，如果大家不知道**麻省理工长毛兔**的话，小红书上可以看到他的主页。你现在怎么自我定位啊？是 **Web Coding** 吗？还是学习博主？

**Speaker 2**: 可能是学习博主，是 **AI** 和 Web Coding 的吧。以后学什么也不知道，就是我也是按照自己的兴趣。其实我就是一个比较愿意学习，然后分享自己的学习过程和成果的人吧。我之所以说自己是普通人、下班后学习 AI，是因为我现在有自己的主业，同时也会在周末和下班的时候去学一些 AI 和 Web Coding 的东西。但是我并没有一个开发的背景，我以前没有做过 **Software Engineering**，做 Development 这种技术开发。

**Speaker 2**: 我的背景的话，本科是学**工业工程**的，然后研究生学的是 **MBA** 和**土木环境工程**。一开始想做环境工程，这种也就是地球的大工程了。这些年的话，就是虽然没有开发背景，但是做过产品经理，做过数据分析，做过数据工程，还有做管理咨询做了很多年。所以有一些比较综合的兴趣和爱好方向吧。最近这半年到一年之间就是比较上头于 Web Coding，觉得非常好玩。所以最近分享的都是这方面的东西。

**Speaker 0**: 而且你分享有一个很特别、或者说专属的开头，就是“你在 AI 上又花了多少钱”，对吧？

**Speaker 2**: 啊，对我现在都已经不数了，因为我觉得就是越来越多，根本停不下来的那种。但是我有一段时间是每周看我花了多少钱，每周也会总结一下最近的学习成果，这周学了点什么东西。那个时候还挺多总结的，最近也不总结了，因为时间太长了就觉得大同小异了。

### AI API 成本控制与算力消耗探讨

**Speaker 0**: 那你还在数的时候，你花了多少钱？

**Speaker 2**: 我数的时候，好像两三千美金吧。但是其实有一半是在买电脑上。那个时候我还是用 Windows，很多人就说你这个不算。但是我买了一个 **Mac**，就是专门为了做 **iOS Development**。我用 Windows 已经很久了，一直没想换，多买了一个电脑就是为了做这个开发。后来我还用它跑那个 **Open Interpreter**，相当于一个单独的机器可以一直开着，不会连我自己的那些信息。所以我把它当做我在这方面的投入吧。

**Speaker 2**: 不算买电脑的话，就是分阶段买不同的东西。现在的话可能一个月二十块钱 **Claude**，二十块钱的 **Lovable**。然后每个月又买 Domain、Hosting 的那些，这些加起来一个月可能就五六十吧，杂七杂八的。我一直在拖延自己没有上 **Claude Max**，我觉得用的不够稳定，就什么时候我好几天连续觉得不够用的话我可能就上了。但是我现在可能一个礼拜只有那么一两天觉得不够用，我就等一等，所以还在拖延。你现在是不是已经上 Max 了？

**Speaker 0**: 那肯定是，还有很多。

**Speaker 2**: 还有很多人推荐搞两三个 Pro 不上 Max，我觉得也可以算是一个过渡。

**Speaker 0**: 我们群里好多人是好几个 Max 加好几个 **ChatGPT Plus**。订阅制的话，一个两百美元的订阅制大概对应的是两三千美元的 API。他们确实要用这么多，所以他们是好几个账号。你看**鸭哥**平均一天半个 billion 的 token。那个**陈然**说他是一天一个 billion 的 token，这个很夸张。但他也是混着用了，本地的也有，然后 **DeepSeek** 有很多，**GLM** 也很多，都不是说一定要用最贵的。

**Speaker 2**: 哎，你身边有很多人公司做 Token Leaderboard 嘛，就是看全公司谁排名前用最多 token 的，我们公司原来也有，身边很多朋友公司也有，最近全撤了。

**Speaker 0**: 对，一个是太贵了。另外一个就是不合理嘛。就是一个 input metric，你怎么可以用这个东西衡量？谁花钱多谁就牛逼嘛，肯定不是这样。但是我也理解，当大家都不用的时候，用这种方式让大家看到，去把它用起来，一定阶段上想要是有道理的。

**Speaker 2**: 有激励型的那种，过渡一下，用多了现在大家就开始控制了。

### 从零入门 AI 工具链：Lovable 与 n8n 实战

**Speaker 0**: 我觉得邀请你过来做直播，就是因为你代表了很多人，你跨过了这个门槛。我对 AI 好害怕，一方面怕它取代我，另一方面我的使用还停留在很简单的方式（跟 AI 聊天）。但是你用过 Agent 工具，做过 Web Coding 以后，使用 AI 能做的事就一下子多很多。想先请你讲讲你的故事，你是怎么跨过这个门槛的？跨过这个门槛有什么好处？

**Speaker 2**: 我不知道大家有没有听过一个叫 **Ready AI** 的产品，是国内的一个团队做的，就是用自然语言可以直接做一个网页的东西。那个时候是二五年春天（二月或三月份），他在湾区做了一次分享会，我就刚好参加了这个活动。我当时看了 Demo 就觉得好有意思。因为我从小到大就一直想做网页，但是我每次看到 HTML、CSS 这一堆东西需要先学的时候我就劝退了。

**Speaker 2**: 我回来后加了 Demo 那个人的微信，要了几个 credit 试一试。然后做了几个网站，觉得非常有意思。在 Lovable 还没火起来的时候，我就先尝到了甜头。之后就开始一发不可收拾，我是从 **n8n** 那些东西开始用的。

**Speaker 0**: 你还经历过这个阶段啊！

**Speaker 2**: 对，我在早些年做管理咨询的时候做过很多 **RPA (Robotic Process Automation)**，里面有 OCR 的东西在里面，我很习惯这种拉线连出来的 no-code 东西。当时做了一个去 Reddit 上抓取某个品牌的二十四小时讨论，每天给你发 email 的流程。也做过 AI 自动回邮件，直到回满了再转人工。早期的时候还是走了很多弯路的。

**Speaker 0**: 你这一个功能在 n8n 上大概要画多长时间？

**Speaker 2**: 如果分两三天的话，每天就两三个小时。刚开始在尝试阶段，链条没有特别长。做完以后发现还是有点麻烦，感觉还是需要 Code。它有时让你 parse string 或者写 JSON。再往后就开始用 Lovable，Lovable 是打开新世界大门的一个东西。它立刻就可以看到你想要的做的事情。我就做了一大堆有的没的的网站。

**Speaker 0**: 你的那个有的没的网站都做了什么？

**Speaker 2**: 有些做过那种 Wrapper，就是天使和恶魔的聊天。你心情不好跟他聊，他会 cheer you up；你飘的时候跟他聊，他帮你 ground 一点。里面就是一个 ChatGPT 的 API。我还做过个人网站，做过猜数字小游戏。

**Speaker 2**: 十二月份的时候，做了一个你可以上传一张照片，我用 Prompt 把它做成一个比较好看的卡通圣诞卡片，发给第三个人。这个是我第一个上线的产品，之前那些都是我自己用的。

**Speaker 0**: 这个产品有人用吗？

**Speaker 2**: 有几千个人来用。这里要感谢一下小红书，当时大家挺给面子来用的。我用 **Resend** 帮我发 email，有一段时间因为人太多了，还把我的账号给冻结了，我还得去解释。

**Speaker 0**: 这件事花了你多少钱？

**Speaker 2**: 开发钱花的不多，最花钱的地方是我自费让大家用我的 API 去做图。一开始没想到有那么多人用，后来应该是烧了三位数的 API，我一直在补。

**Speaker 0**: 三位数还好一点。几千人用已经是一个比较火的产品了，而且图像生成比较贵。大家老是担心用 AI 刷出来天文数字，其实也就是一百多美元。

**Speaker 2**: 对，我也不会让它自动 load 钱，就是手动提醒我再加。但是我从这里学到的就是，AI 时代的产品实时的烧 token，没有 SaaS 那么 scalable。以前 YC 投的产品基本上不花钱，像 Uber、Airbnb，十个用户和一百个用户成本差别不大。但是在 AI 时代，每个用户都是实打实的 token 钱。

**Speaker 0**: 二四年的播客里，**Anthropic** 的 CEO Dario 说现在的 AI 公司更像制造业，是有成本的。他们一定要规划好 token 未来的 demand。

**Speaker 2**: 现在很多人说趁着大公司 subsidize 我们压价，他们烧的钱其实比我们每个月二十块钱还多。相当于他们用低价来弥补我们了。

**Speaker 0**: 我不认同这个想法。OpenAI 讲过他们的 gross margin 是百分之七十五。绝大多数人二十块钱是根本用不到多少的，API 也是有很高的利润率。他们有研发和人工成本，但从 host token 角度来说并没有赔钱。

### 独立开发者的产品上线与市场验证 (PMF)

**Speaker 2**: 我觉得它给我最大的 learning 就是产品上线的时候，会有很多问题出现。账号被封、数据库出问题，这种线上救火对于新手独立开发者还是有一点紧急的。那段时间我每天都在凌晨三四点才睡觉。其实比起上线出了小 Bug，我更担心的是发出去没有人用，没有水花。如果有人用，并在评论区立刻给你 Feedback，你马上去改的话，这是一个非常健康的 loop。

**Speaker 0**: 所有的脑子里想的东西都没什么意义。你上线了以后就会发现想的都是错的。

**Speaker 2**: 是的，我很欣赏湾区的投资大佬和科技大佬们朴实的想法：我不能预判未来，所有的事情都是一个 experiment。带着做实验的心思去。

**Speaker 0**: 你在上线之前不会担心安全漏洞吗？很多人对 Web Coder 的诟病，就是他们做的东西不是 product-ready 的。

**Speaker 2**: 其实没有这个担心。如果真的是我修不了，我用 AI 也修不了，可能会打电话给一些学 CS 的朋友求助。上线前我会让 Lovable 做一些 security scan，用 Claude 过一遍 code，自己找朋友测试没问题就上线。最重要的事情如果都已经准备好了，就让真实的反馈给你机会去尝试。比起修 Bug，我更担心没人用。我最近也在和创业团队做产品，遇到一个两难：我能实现 solution 的东西，很难找到客户去买；我能解决的问题，产品又很难做出来。找 **Product Market Fit (PMF)** 比想象的难。

**Speaker 0**: 超难，所有的创业不成功全都是因为没有 PMF。有了以后就会被推着走。

### 自媒体运营：内容系统与爆款逻辑

**Speaker 0**: 回到你的小红书，你的涨粉路径是什么样子的？什么时候开始爆起来的？

**Speaker 2**: 二二年我入学麻省理工的时候开始做小红书。一开始想拍 vlog，周末剪一下发视频，维持了半年。到我毕业两年大概是一点八万粉丝。

**Speaker 0**: vlog 做到一点八万很厉害了！也是因为麻省理工的招牌吧。

**Speaker 2**: 我觉得是的。后来也有黑粉攻击说“麻省理工的还不会写代码”。我肯定是借了这个品牌的力来建立自己的品牌，那就要承担它的反噬后果。毕业入职后停了半年。二五年春天开始学 Web Coding 时，又开始比较规律地发，慢慢做到了十万粉丝。

**Speaker 0**: 从三万怎么涨到十万的？更新频率是多少？

**Speaker 2**: 频率加上去了。有三四个月是每周发三个视频。我会在一个 Excel 里写好这周和下周的 Topic，每周日坐下来把三个视频全录了。因为女孩子要洗头发化妆比较麻烦，周日就是我的录制日，然后分期去剪辑发。

**Speaker 0**: 你发的 Web Coding 东西基本上都是过千赞。

**Speaker 2**: 有段时间每天打开都是 99+。时机也是一方面，那时 Web Coding 越来越火，我发的早一点，尽量用比较直白的语言真诚分享。门槛没有很高，大部分人都能听懂。有些特别厉害的大神讲得太 technical，很多人就看不下去。

**Speaker 0**: 你的视频非常清晰，节奏点给到正确的信息，ROI 非常高。这是怎么做到的？

**Speaker 2**: 观众缘可能就是性格和长相，像个朋友一样。表达方式上其实大量依靠后期剪辑，我一句话可能会讲很多次。一个三分钟的视频，我可能录了二十分钟，然后剪出来的。

**Speaker 0**: 如果你能全说中文当然好，但是在海外久了会不可避免地中英夹杂。我们的脑子在想概念时并不是语言，映射到语言上如果强制用中文，思路就会慢一半。

**Speaker 2**: 我不会去写逐字稿，读逐字稿会让我们变得更机械化。我还是希望能像朋友一样自然对话。我会写大纲 Bullet points，时不时看一眼有没有漏掉。我最近连字都不打了，用 Voice 工具代替写作输入。

### 生产力工具流与避免“工具内耗”

**Speaker 1**: 我给你展示一个，我们社区做的 **Super Linear Academy** 语音工具（v.superlinear.academy）。完全尊重隐私，而且可以和 Apple 生态打通，手机上语音录入，电脑剪贴板直接就有了。

**Speaker 0**: 你给大家介绍一下你现在常用的工具？时间成本是怎么解决的，大家觉得这么多工具没时间去试？

**Speaker 2**: 这里我要引荐 Claude 经常用的叫 **Progressive Disclosure**（渐进式展示）。市面上有新工具时，我花个一两分钟看它是干嘛的，知道它的 one-liner。我先不用，除非我现在刚好需要它提升效率，我才会花时间学并采用。如果好用就留下，不好用就淘汰。我不会横向深度对比研究工具。工具淘汰太快了。

**Speaker 0**: 对，工具不重要，Mindset 大于习惯，习惯大于方法，方法大于工具。现在三分钟学会 Claude 就能拿到最厉害的模型，后面都是方法。

**Speaker 2**: 我现在的核心工具流：先在 Claude 的对话框（手机或电脑版）里聊想法和功能，聊清楚后让它生成 Development Plan、Design Brief 等 MD 文档。然后把文档扔进 Cursor 或者 Claude Code 终端里面，按照计划一步步开发代码，最后让设计师或 Lovable 辅助页面设计。

**Speaker 0**: 我其实是把在聊天里形成的内容，直接就在 Cursor 里操作了。很多同学没有意识到把聊天生成工作文档再喂给代码工具，会极大提升效率。

**Speaker 2**: 我作为小白打开终端 Terminal 还是会有心理摩擦的，跳到 Terminal 模式是一个大转变。但是在聊天界面我可以一天聊十个主意。另外，关于需求，并不是你要做多大的事情，哪怕是给自己做个记录健身或喝奶茶次数的简单工具网页，只要能解决你生活中的痛点，每天帮你省两分钟，这就足够支撑你从动手构建中获得反馈与进步。边做边学是最好克服恐惧的方法，千万不要陷入到不停学习纯理论知识的陷阱中。
