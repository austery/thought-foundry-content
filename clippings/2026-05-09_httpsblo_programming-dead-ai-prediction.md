# 编程已死？Claude Code 之父的激进预言

AI资讯

·

2026年5月9日

·

66 次阅读

·

约 7 分钟

Boris Cherny 是 Claude Code 的创造者。

他在 Sequoia Capital 的一场活动上问台下的人："谁写代码是 100% 靠 AI 的？"

举手的人不多。

然后他说："对我来说，就是 100%。"

这句话很容易被误读成吹牛。

但我觉得这是目前关于 AI 编程最诚实的一句话——不是"AI 能辅助你"，不是"AI 能提升 10 倍效率"，而是：编程这件事，作为一种人类手工活，已经结束了。

他是这个工具的作者，所以他比任何人都更早到达那个终点。

## 一个"意外"产品的诞生

有意思的是，Claude Code 一开始几乎是个失败项目。

2024 年底，Boris 加入了 Anthropic 内部的一个小型孵化团队。

他们看到了一个"产品 overhang"——模型的能力已经超过了现有产品所能捕捉的边界。

当时最先进的 AI 编程工具还是代码自动补全，按 Tab 一行一行地来。

Boris 觉得这太保守了。为什么不让 AI 直接写全部代码？

于是他开始做 Claude Code。

结果前六个月，几乎没人用。

发布之后，也没有爆炸式增长。

真正的转折点是 Opus 4 上线的那一天。

那之后，每一代新模型都带来了一次新的增长拐点。Opus 4、4.5、4.6、4.7……模型能力的每一次跃升，都让产品重新活一次。

这给了我一个很重要的启示：有些产品，不是在等用户教育，而是在等模型进化。

你在为下一个模型建造，而不是为当前模型优化。这是一种完全不同的产品哲学。

## 他现在到底怎么工作？

Boris 分享了他的工作方式，坦率地说，有点颠覆认知。

他主要用手机工作。

打开 Claude 移动端，左侧有一个 code tab，里面同时跑着五到十个会话，每个会话下面又有多个 agent。

他说，平时大概有"几百个 agent 在运行"，晚上做深度任务的时候，能跑到"几千个"。

上周有一天，他完成了 150 个 PR。

这已经不是人类程序员的工作节奏了，这是一个指挥官在调度一支代码军队。

他特别推崇的一个功能叫 Loop——让 Claude 用 cron 定时任务循环执行某件事。他现在有几十个 loop 在后台持续运行：

一个在盯着他的 PR，自动修 CI、自动 rebase

一个在保持 CI 健康，发现 flaky test 就修

一个每 30 分钟从 Twitter 抓取反馈并聚类

他说："我感觉 loop 就是未来。"

Anthropic 内部也是一样。

所有人的 Claude agent 在跑的时候，会通过 Slack 和别人的 Claude 实时沟通，协调未知问题。

没有人再手写 SQL，没有人再手写业务代码，全部由模型生成。

## "编程已解决"意味着什么？

Boris 很清楚这个说法的边界。

对他来说是"已解决"，因为他用的是 TypeScript + React，这是模型训练数据最丰富的技术栈，模型驾轻就熟。

对于那些有复杂历史代码库、冷门语言、奇特框架的团队来说，还没到那一步。

但方向是确定的：等下一个模型就行了。

这是他反复强调的一句话。

今天解决不了的问题，等下一个模型基本都会解决。这听起来像一句空话，但在 Claude Code 的成长史里，它被反复验证了。

## 印刷术时刻

我最喜欢他讲的一个类比。

他说，今天编程的处境，和 1400 年代欧洲识字的处境非常像。

印刷术发明之前，欧洲只有大约 10% 的人识字。

识字者受雇于国王和贵族，读写是一种专业技能，是少数人的特权。

印刷术出现后，50 年内欧洲出版的书籍比过去一千年的总和还多，书的价格跌了 100 倍。

再之后几百年，全球识字率从不到 10% 涨到了 70% 以上。

编程将会经历同样的历程，但会快得多。

他说，未来写会计软件最好的人不是工程师，而是会计——因为他们懂业务，而编程只是工具。这个逻辑放在任何领域都成立。

软件的民主化，不是让所有人都去学编程，而是让所有人都能用自然语言表达需求，让 AI 完成实现。

## SaaS 末日？还是黄金时代？

有人问他：当写代码的成本降低 100 倍，软件产品的价值会怎么变？SaaS 会不会完蛋？

他给出了一个很有框架感的回答。

他引用了 Acquired 播客里提到的"七种商业护城河"，说 AI 时代，有些护城河会变弱：

切换成本变弱——AI 可以帮你从一个工具迁移到另一个工具

流程壁垒变弱——AI 可以快速理解并优化任何工作流

但另一些护城河依然坚固：

网络效应

规模经济

稀缺资源

这些不会因为 AI 写代码变便宜而消失。

同时，他认为创业公司迎来了黄金时代。

大公司面对 AI 的最大阻力是内部流程重塑、人员培训、文化转型——这些都是慢变量。

而初创公司从第一天就能以 AI-native 的方式构建，没有历史包袱，没有内部阻力。

你的起点就是大公司花五年才能到达的终点。

## 技术领先的真相

最后有人问他：Anthropic 内部用的模型是不是比外面强很多？领先几个月？

他的回答出乎意料。

他说，在模型层，其实没什么领先——他们用的是 Opus 4.7，外部用户用的也是。

真正的领先，在组织层。

Anthropic 把整个公司的工作流重塑了一遍。

所有人用 Claude，所有代码由模型写，所有 SQL 由模型生成，agent 之间通过 Slack 实时协作……这种组织形态，才是真正难以复制的东西。

技术是平台，开放给所有人用。

但如何把组织重新设计成 AI-native 的样子，这是个每个公司都需要自己探索的问题。

有意思的是，他说自己 2026 年迄今为止一行代码都没亲手写过。

一个写了一辈子代码、还写过 TypeScript 教科书的工程师，亲手宣告了"手写代码时代"对他个人的终结。

这不是悲剧，他说得兴高采烈。

因为终结的只是一种手段，释放出来的是更大的创造力。

就像印刷术没有让写作消失，只是让写作从贵族的特权变成了普通人的日常。

编程也会如此。
