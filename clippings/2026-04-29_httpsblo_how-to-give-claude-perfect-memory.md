# 如何让 Claude 拥有完美记忆

> 原文链接： How to Give Claude Perfect Memory (complete guide)

说实话，Claude 默认的记忆功能基本没什么用。

它经常忘记上下文，你得不停重复解释，即使解释了，它还是记不住。

大多数人已经忍受这些问题好几个月了，却不知道其实有更好的办法。

我每天都在用 Claude。它在我 Mac 上的使用时长，比任何其他应用都长。

正因为用得这么频繁，我不能接受最常用的工作工具随机忘记重要信息和上下文。

为了让 Claude 尽可能靠谱，我深入研究了所有可能的记忆解决方案。

好消息是，研究很成功。

## 三层记忆系统

我发现了三个"层级"的记忆系统，它们让 Claude 变得强大了很多。

第一层：简单易用，能满足 90% 以上 Claude 用户的需求。

第二层：需要大约 60 分钟设置，但会彻底改变 Claude 的工作方式。

第三层：则把 Claude 变成一个自我进化的第二大脑，基于你所有的数据训练。

今天这篇文章，我会公开这三个系统，以及我如何让 Claude 变得更聪明。

无论你是新手还是重度用户，相信你都能找到适合自己的系统。

## 第一层：基础设置（4 个快速优化）

这是每个人都应该开始的地方，只需几分钟设置，就能立即改善你和 Claude 的每次对话。

### 1. 清理你的记忆页面

现在就去设置 → 记忆。

这是 Claude 里最被忽视的页面，大多数人从来没打开过。

你会看到 Claude 存储的关于你的所有信息，偏好、事实、习惯、工作风格等，这些都是从你所有对话中被动积累的。

如果不管理，你的记忆很快就会堆满垃圾。

解决办法很简单：

通读这个页面的所有内容，删除过时、不准确或无关的信息。

然后，手动添加你真正希望 Claude 永久记住的上下文。

这里保持基础就好，你的角色和基本偏好，我们很快会深入构建更具体的系统。

### 2. 填写项目说明

如果你使用 Claude 项目功能（你应该用），就需要填写项目说明字段。

我的建议：

为所有最常用的工作流创建项目，然后把所有上下文用语音输入到 Google 文档，再为每个项目上传 PDF 文件。

### 3. 对话中直接告诉 Claude 记住什么

这是列表中最简单的记忆技巧。在对话中，直接告诉 Claude 你希望它记住什么。

```
<code>"记住我永远不要 [x]"
"记住我的角色是 [x]"
"用 [x] 更新你的记忆"
"记住我更喜欢 400 字以内的回复"</code>
```

Claude 会立即把这些存储到记忆中。你也可以让它忘记东西：

```
<code>"忘记我提到的 [x]"</code>
```

### 4. 记忆导入和导出

如果你一直在用 ChatGPT（或其他大语言模型），并且已经积累了大量上下文。

你有两个有效的选项来转移上下文：

方案 a） 你可以告诉 ChatGPT 你要换平台，让它生成一个记忆导出文档：

```
<code>"我要把这个项目切换到 Claude，给我一个总结文档……"</code>
```

方案 b） 你可以使用 Claude 的导入/导出功能

在设置 → 记忆中，你可以从其他大语言模型导入完整数据。

这四个快速调整对 90% 以上的人来说已经足够。

但下一部分是为那些想要真正系统化、把 Claude 提升到新高度的人准备的。

## 第二层：基于文件的记忆架构

第一层解决了基本的记忆问题。

第二层构建了更强大的东西：一个基于文件的记忆架构，存储在你的电脑上，自动加载到 Cowork 和 Claude Code 中。

概念很简单：

不再向 Claude 提示上下文，而是把所有上下文存储在 Claude 可以访问的 .md 文件中。

你也可以把这些 Markdown 文件用在任何大语言模型或 AI Agent系统。

### 构建你的 Claude 主文件夹

在桌面新建一个文件夹，命名为 "Claude 主文件夹"，在里面创建这四个 Markdown 文件（Claude 可以帮你做）：

1. Instructions.md（指令文件）

这个文件告诉 Claude 你所有的规则和指令。

> 示例结构：你是谁你做什么规则好的输出是什么样的

重要的是要包含："随着时间推移，用我的偏好更新 Memory.md"

这一行至关重要，它让 Claude 在第二个 Markdown 文件中创建你数据的持续记忆日志。

2. Memory.md（记忆文件）

这是 Claude 的"大脑"，会随着时间不断更新。

> 示例结构：偏好修正模式决策

现在，只要你说 "别再用破折号" 之类的话，Claude 就会进入记忆文件并更新它。

3. Context.md（上下文文件）

> [x] 项目的具体上下文文件。

这个 Markdown 文件里的内容会根据你的具体项目而变化。

你也可以创建一个通用的 "业务上下文" 或 "生活上下文" Markdown文件。

4. Archive-Guide.md（存档指南）

这个纯粹是保护性的，但值得做。

Claude 会在你工作时自动更新你的记忆文件。偶尔，它会错误覆盖某些内容，或做出你不想要的更改。

没有备份系统，那些上下文就丢失了。

解决办法很简单：

每周一次，把包含 Instructions、Memory、Context 和其他所有内容的整个主文件夹复制到一个 Claude 无法访问的单独存档文件夹中，并标注日期。

如果出现问题或被错误覆盖，你可以从存档中恢复。

### 最终产品应该是什么样子

这是四个 Markdown 文件的最终产品结构。

### 让 Claude 帮你创建

如我上面提到的，你可以让 Claude 帮你创建这个系统。

只需创建一个名为 "Claude 主文件夹" 的新文件夹，将它附加到新的 Cowork 聊天中，然后粘贴这个提示词：

```
<code>进入我连接的工作区中的 "Claude 主文件夹"，在里面创建这四个 Markdown 文件：

Instructions.md，包含以下部分：你是谁、你做什么、规则、好的输出是什么样的，以及一行告诉 Claude 随时间更新 Memory.md 的内容。

Memory.md，包含以下部分：偏好、修正、模式、决策和个人上下文。预填充占位符示例，这样我就知道该添加什么。

Context.md，包含以下部分：关于这个项目/业务、受众、关键人物和协作者、活跃项目和优先事项、工具和技术栈、重要背景/历史。使用模板格式和占位符，我可以填写。

Archive-Guide.md，一份分步指南，解释为什么要存档、如何每周执行（复制文件夹、用日期重命名、移动到 Claude 无法访问的地方）、包含什么、如果出问题如何恢复，以及在哪里存储备份。</code>
```

### 如何使用这个系统

系统建好了，现在怎么用呢？

每次在 Cowork/Claude Code 中工作时：

你可以附加主文件夹，Claude 就会把它当作迷你记忆数据库使用。

它会编辑记忆 Markdown 文件，给你留下一个可以附加到任何大语言模型、新聊天或 AI Agent的东西。

你也可以手动更新 .MD 文件，并在主文件夹内为特定项目创建新文件夹。

接下来要在第三层展示的内容，会把这个系统推得更远。

## 第三层：AI 第二大脑（高级选项）

坦白说，不是每个人都需要。

它确实需要一些初始设置和持续维护，但对于那些构建它的人来说，这确实是 Claude 高级详细记忆系统的最佳选择。

根据你的工作方式，有两个选项。

第一个更简单，适合那些想要 "简单" AI 第二大脑的人。

第二个更高级（但整体更好），需要 1 到 2 小时的专注构建（别担心，我会全程指导）。

记住：

要让你的 AI 第二大脑记忆库有效，你确实需要花时间维护和更新你的数据库。

### 选项 1：Claude x Notion

将 Claude 连接到 Notion 是你在 5 分钟内能做的最高杠杆的事情。

前往 Claude → 设置 → 连接器，然后启用 Notion 连接器。

连接后，Claude 可以在任何聊天中直接读取你的 Notion 工作区。

现在你所有的任务、CRM、笔记、表格等都可以被 Claude 访问和编辑。

我建议：

创建一个新的 "记忆数据库"，在那里存储所有 AI 偏好、规则和重要的 AI 上下文。

当你和 Claude 工作时，可以说：

```
<code>"把这个发送到我的 Notion 记忆数据库"</code>
```

然后你可以通过 CSV 文件或使用 Notion MCP 连接器，将这些 Notion 数据导出到其他大语言模型或 AI 平台。

这个设置类似于我在第二层介绍的内容，只是现在你有了 Notion 内置的看板视图、待办事项列表等漂亮的可视化界面，并且解锁了额外的功能。

我个人不常用这个设置（下面的选项 2 更好），但我偶尔会用它来发送和存储我在 Claude 中使用的有价值的超级提示词：

### 选项 2：Claude x Obsidian

Obsidian 是一个将所有内容存储为纯 Markdown 文件的工具，存在你的电脑上，是连接 Claude 并构建第二大脑的可靠方式。

步骤 1：下载 Obsidian

前往 obsidian.md 下载应用。

创建一个新的 Vault（把它想象成一个简单的桌面文件夹，Claude Code 将在其中存储和访问你的数据）。

步骤 2：在 Claude Cowork/Claude Code 中选择 Vault

打开 Claude 桌面应用，点击 "选择文件夹"。

指向你的 Obsidian Vault 文件夹。Claude 现在对里面的所有内容都有直接的读写访问权限。

步骤 3：使用 Andrej Karpathy 的大语言模型知识库系统提示词

将 Andrej Karpathy 的大语言模型知识库系统提示词粘贴到聊天框中。

这是告诉 Claude Code 如何随时间构建、维护和演化你的维基的指令集。

提示词在这里可以找到：Andrej Karpathy 的知识库系统提示词

步骤 4：导入你的数据

放入任何现有的笔记、CSV 文件、文章导出或 Notion 导出，开始填充你的第二大脑。

Claude 然后会摄取每个来源，提取关键信息，并将其整合到一个不断演化的记忆维基中。

最终产品是一个 AI 第二大脑知识维基。

### 应该选择哪一个？

Notion = 快速、简单的选项

Obsidian = 本地存储，你希望 Claude 有深入理解（这是我个人发现的最高级的记忆系统）

Claude Code + Obsidian 终极指南（构建 AI 第二大脑）

Claude Code + Obsidian 是我用过的最强大的 AI 组合。我真的构建了一个 AI 第二大脑，包含我思考、阅读、写作、在线研究的所有内容，甚至更多……

## 总结

这就是我构建 Claude 记忆系统的三个层级，让它变得更加靠谱。

再说一次：

第一层是基础设置，能产生立竿见影的效果（对大多数人来说足够了）

第二层需要一些设置，但非常有价值

第三层会彻底改变你使用 Claude 的方式
